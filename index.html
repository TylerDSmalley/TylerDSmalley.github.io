Elysium Escapes

Visit Elysium Escapes Source Code Presentation

Elysium Escapes is a mock travel site catering to high end clients. It was built by Ileana Luts, Stephan Needham and I. This was an integration project for my web developement class. As such, this project touches all aspects of a working web page from front to back. The site is fully functional in a mock setting.

User's can browse destinations, read travel stories from other customers and review their past orders. All of these items are populated dynamically from the database. Visitors can contact the site owners through a form on the contact us page, which will store their message in the DB. A user can register and log into an account with the site. If they forget their email, a password recovery link is sent to their email address. Finally, they can select a destination, select their desired dates for their trip, and the number of adults/children that will be traveling. Submitting this form sends a realtime request to the Booking.com api which will return a selction of high end resorts that match their criteria. The results are displayed in a series of modals and within each modal the user can browse photos that the api sent for the given hotel. When the user is ready, they may book a trip. To handle payment processing, the Stripe api was used. As this is a mock site, no actual transactions are permitted, so we use Stripe's provided test card numbers. Upon completion, the user is taken to an order summary page. At the same time, a confirmation summary is sent to the user's registered email address. The user can review their past trips from the user menu where they may also post their own travel story that will be published to the site blog. The site was built using various technologies. Html was written using the twig template engine. For CSS styling, a mix approach included the bootstrap 5 library as well as my own custom style sheets. PHP was handled using the Slim framework coupled with MeekroDB to streamline the Database handling and MySql/MariaDB was used to create the database itself.

ITems

Visit ITems Source Code Presentation

ITems is a mock Employee management system built by Ileana Luts and I using Java Spring Boot, Java Spring Security, thymeleaf templating engine, Bootstrap 5 and MySql Locally/ PostgreSQL hosted on Heroku. We were given about a week to build an initial proof of concept for our Java/Programming III class. To see where this project would be headed given a larger timeframe, please refer to the project presentation below.

Being an EMS, anyone attempting to visit the app will be automatically redirected by Spring Boot Security to the login portal. Only current HR employees with the role-MANAGER can add new employees to the system. Upon onboarding, the new employee is sent an email with their initial log in credentials, inviting them to log in and update their profile. The link provided in the email will push them straight to the update page to ensure their first step will be to submit their personal info to HR.

Upon successful sign in, the employee is taken to the dashboard where they can view three cards: Employee Profile, HR requests, and Employee documents. After being clicked, these three cards expand to show more detailed information as well as buttons to change/submit new info to HR. For example, an HR request can be submitted from the dashboard requesting time off or location transfer. Once the transfer is submitted, the HR manager can view the request from the Admin dashboard and choose to approve or deny. The status of the request is displayed for the Employee to see in the HR request card dropdown
